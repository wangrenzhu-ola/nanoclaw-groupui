## 1. 排期方法说明

1. 本排期单位为“对话轮次（Round）”，非自然日工期。
2. 每轮交付定义：本轮开发功能完成 + 本轮测试通过 + 产出验收记录。
3. 每轮结束 MUST 通知你进行人工验收；未验收通过不得进入下一轮。
4. PMO方法参考 `find-skill` 检索到的 `project-manager` 与 `technical-launch-planner` 角色实践。

## 3. 每轮输入输出标准

### 3.1 输入标准

1. 上一轮人工验收结果为“通过”。
2. 上一轮遗留缺陷已完成分级与处置计划。
3. 本轮任务、测试、回归边界已冻结。

### 3.2 输出标准

1. 代码与文档变更已提交（不要求Git commit，由执行Agent管理）。
2. 本轮对应测试用例执行完成并留痕。
3. 发布说明包含：完成项、风险项、待验收项。
4. 向你发送人工验收通知，附本轮证据清单。

## 5. 风险与应对

1. 轮次膨胀风险：若单轮范围超过3个P0能力，MUST 拆分新轮次。
2. 测试阻塞风险：若P0用例未通过，MUST 回退到当前轮次修复，不可推进。
3. 需求变更风险：新增范围 MUST 先更新 OpenSpec 再纳入新轮次。
