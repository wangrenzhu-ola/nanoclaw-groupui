## 1. 测试范围与目标

1. 覆盖 NCW-FR-001~028 的全部功能、异常、边界与治理能力。
2. 每个测试用例 MUST 可复现、可判定、可追踪。
3. 执行结果 MUST 回填到测试跟踪表与缺陷表。

## 2. 测试环境基线

1. 部署方式：Docker Compose 私有化部署。
2. 浏览器：Chrome / Edge 最近两个稳定版本。
3. 数据：预置公开频道、私有频道、1v1 DM、Group DM、线程与审计样本。
4. 角色：工作区管理员、频道管理员、普通成员各 1 个以上。

## 3. 用例设计规范

1. 用例命名：`TC-NCW-<模块>-<序号>`。
2. 优先级：P0/P1/P2。
3. 执行格式：Arrange-Act-Assert。
4. 通过标准：Expected Result 全部满足即 Pass。

## 5. 缺陷分级标准

1. P0：安全漏洞、数据丢失、核心链路不可用。
2. P1：主功能受损但有临时绕行方案。
3. P2：局部功能异常或边界失败。
4. P3：体验、文案或低风险一致性问题。

## 6. 交付与验收门禁

1. P0 用例通过率 MUST 为 100%。
2. P1 用例通过率 MUST ≥ 95%。
3. 未关闭 P0 缺陷 MUST 为 0。
4. 每轮交付后 MUST 通知人工验收并记录结论。
5. R2 交付前 MUST 通过 `TC-NCW-LLM-CONN-001` 与 `TC-NCW-LLM-CONN-003`。
